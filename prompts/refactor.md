# Safe Refactoring Prompt

Refactor this code while:

## Constraints
- Preserve all existing behavior
- Keep the public API unchanged
- Maintain or improve test coverage
- Make incremental, reviewable changes

## Goals
- Improve readability
- Reduce complexity
- Remove duplication
- Follow project conventions

## Process
1. Identify what to change
2. Explain the refactoring approach
3. Show before/after for each change
4. Highlight any risks or edge cases

## Do Not
- Change behavior without explicit request
- Remove or weaken tests
- Break existing interfaces
- Introduce new dependencies without justification
