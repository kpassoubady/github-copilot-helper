# Documentation Prompt

Generate documentation for this code:

## Include

**For Functions/Methods:**
- Brief description of purpose
- Parameters with types and descriptions
- Return value description
- Example usage (if helpful)
- Exceptions/errors that can be thrown

**For Classes:**
- Class purpose and responsibility
- Key methods with brief descriptions
- Usage example
- Related classes (if relevant)

**For Modules/Files:**
- Module purpose
- Key exports
- Dependencies
- Usage patterns

## Style
- Use the project's existing documentation style
- Be concise — avoid redundant information
- Focus on "what" and "why", not implementation details
- Use proper JSDoc/Javadoc/docstring format for the language
